## 🤔 为什么做这个

假设你重生了。这次你是大厂 VP，年薪 200 万，手下一堆人。

你开完会说了句"方向对了，继续"。总监接过去换了个格式转发。政委加了句"符合价值观"。项目经理写了份周报。四个人忙了一圈，干活的还是那个 P6。

每个大厂程序员都知道这件事，但没人能量化。

OrgMirror 做的事情很简单：把这些角色全部换成 AI Agent，按真实组织架构跑一遍任务，然后看报告。

每个 Agent 消耗了多少 Token，产出了多少新信息，影响了下游多少决策，能不能直接跳过。白纸黑字，一目了然。

三套架构跑同一个任务。字节最快，阿里最慢，腾讯最浪费。重生成哪家的高管最划算？答案可能让你不想重生了。

<details>
<summary><b>🔍 为什么"效率审计"是杀手锏？</b></summary>

<br>

现在所有 Multi-Agent 框架都在吹"多 Agent 协作"，但没人问一个问题：**这些 Agent 真的都需要吗？**

OrgMirror 的审计引擎会追踪四个核心指标：

**信息增量**：Agent 的输出里，有多少是新增的，多少是把上游的话换了个说法重复一遍。VP 说"方向对了，继续"，信息增量约等于零。

**决策影响**：这个 Agent 的输出是否真正改变了下游 Agent 的行为。如果移除它，下游该干嘛还干嘛，那它就是纯开销。

**可跳过性**：综合信息增量和决策影响，算出一个分数。高于 60% 就可以考虑裁掉。

**传话筒检测**：输入和输出的文本相似度超过 85%，直接标红。这种 Agent 就是把 A 的话转述给 B，自己啥也没加。

跑一遍阿里模式你就知道：合伙人委员会、VP、总监、项目经理，四个角色加起来消耗了 43% 的 Token，但信息增量总共不到 15%。

## ⚔️ 三架构效率PK

同一个任务，三套架构跑完的预期结果：

| 指标 | 字节 · 扁平 | 阿里 · 中台 | 腾讯 · 赛马 |
|------|:---:|:---:|:---:|
| **Agent 数量** | 8 | 13 | 12 |
| **总 Token** | ⭐ 最少 | 最多 | 中等 |
| **传话筒数量** | 0~1 | 3~5 | 1~2 |
| **管理层开销** | <15% | >40% | ~25% |
| **赛马浪费** | 0% | 0% | ~30% |
| **执行效率** | ⭐ 最高 | 最低 | 中等 |
| **产出质量** | 高 | 高 | ⭐ 可能最高 |

字节在简单任务上碾压。8 个 Agent 里有 6 个是执行层，几乎没有管理开销。

阿里最慢。13 个 Agent 经过四层管理传递，中台响应还要额外等待。政委审查在任何场景下都是纯开销。

腾讯最浪费，但可能出最好的结果。两个 BG 做同一件事，赢家的质量确实更高。代价是一半人白干了，30% 的 Token 扔进了垃圾桶。

> 重生选哪家？字节效率最高但没有管理岗让你摸鱼。阿里管理岗多但全是传话筒。腾讯最刺激——赢了是功臣，输了整个 BG 白干。

## 🔧 审计引擎怎么算的

不是拍脑袋打分，是有量化依据的。

### 信息增量

用 SequenceMatcher 计算输入和输出的文本相似度。相似度越高，说明 Agent 只是在复述上游的话。VP 说"方向对了，继续"，跟输入内容几乎一样，信息增量趋近于零。

### 决策影响

追踪下游 Agent 是否采用了此 Agent 的输出。如果下游 Agent 的输入里大量包含此 Agent 的输出内容，说明它的产出被真正使用了。如果没有，说明它说了等于没说。

### 可跳过性

综合信息增量和决策影响，加上是否被标记为传话筒。分数越高，说明这个 Agent 越可以被直接跳过。高于 60% 的 Agent，在报告里会被建议裁掉。

### 贡献分类

| 评级 | 条件 | 说明 |
|------|------|------|
| 🟢 核心 | 执行层 + 信息增量>40% | 真干活的 |
| 🟡 辅助 | 信息增量>30% | 有贡献但非核心 |
| 🟠 传话筒 | 信息增量<15% | 输入≈输出 |
| 🔴 纯开销 | 不满足以上任何条件 | 白花 Token |
